# 识别参数配置说明

本文档集中说明当前自动刷新流程里所有和识别、点击、等待有关的参数。后续如果游戏 UI、分辨率或模板图片变化，优先从这里定位需要校准的代码常量。

## 坐标基准

项目当前同时使用两套坐标基准：

| 用途 | 基准分辨率 | 代码位置 | 说明 |
|---|---:|---|---|
| 价格 OCR、商品行购买区域 | `1920 x 1080` | `app/services/price_ocr_service.py` | 来自 E7AutoShop 价格识别方案；ADB 模式强制校验该分辨率，鼠标模式按窗口截图缩放。 |
| 购买弹窗确认、顶部获得提示、商店刷新确认 | `1280 x 720` | `app/services/shop_flow_vision.py` | 来自 e7Helper 流程说明；两种模式都会缩放到实际截图或窗口坐标。 |

鼠标模式窗口点击坐标会额外加上窗口左上角偏移：

```text
screen_x = window.left + base_x * window.width / base_width
screen_y = window.top  + base_y * window.height / base_height
```

## 价格 OCR 参数

代码位置：`app/services/price_ocr_service.py`

| 参数 | 当前值 | 说明 |
|---|---:|---|
| `BASE_WIDTH` | `1920` | 价格 OCR 坐标基准宽度。 |
| `BASE_HEIGHT` | `1080` | 价格 OCR 坐标基准高度。 |
| OCR 引擎 | `RapidOCR()` | 进程内复用一个实例。 |
| 预处理放大倍数 | `3x` | 小价格区域放大后再 OCR。 |
| OCR 文本清洗 | 只保留 `0-9` | 不做 `O -> 0` 等纠错，避免误买。 |

| `item_1` | `top` | `(1660, 140, 1860, 190)` | `1600-1850` | `222-270` |
| `item_2` | `top` | `(1660, 350, 1860, 410)` | `1600-1850` | `440-485` |
| `item_3` | `top` | `(1660, 570, 1860, 630)` | `1600-1850` | `660-705` |
| `item_4` | `top` | `(1660, 790, 1860, 850)` | `1600-1850` | `875-918` |
| `item_5` | `bottom` | `(1660, 680, 1860, 735)` | `1600-1850` | `770-811` |

## 购买确认参数

代码位置：`app/services/shop_flow_vision.py`

| 业务阶段 | 参数 | 当前值 | 说明 |
|---|---|---:|---|
| 购买弹窗确认 | `PURCHASE_DIALOG_ITEM_REGION` | `(430, 160, 850, 470)` | 点击购买后，在购买弹窗中查找同一商品图标。 |
| 购买弹窗确认 | `CONFIRM_BUTTON_POINT` | `(798, 504)` | 弹窗商品图标确认成功后点击确认。 |
| 购买弹窗确认 / 顶部获得提示 | `SCREEN_CENTER_POINT` | `(640, 360)` | 等待失败后的恢复点击点。 |
| 顶部获得提示 | `REWARD_TOAST_ITEM_REGION` | `(531, 48, 649, 155)` | 确认购买后，在顶部获得提示区域查找同一商品图标。 |
| 图标匹配 | `ICON_MATCH_THRESHOLD` | `0.70` | 多尺度商品图标模板匹配阈值。 |
| 图标匹配 | `ICON_TEMPLATE_SCALES` | `0.65-1.45` | 模板缩放倍率集合，用于抵抗截图或窗口缩放导致的图标尺寸差异。 |
| 购买弹窗确认 | 单次等待 | `5s` | 每 `0.3s` 截图匹配一次。 |
| 顶部获得提示 | 单次等待 | `5s` | 每 `0.1s` 截图匹配一次。 |
| 购买弹窗确认 / 顶部获得提示 | 最大重试 | `3` 次 | 三次失败抛出异常并终止本次运行。 |
| 顶部获得提示 | 成功后等待 | `1.5s` | 顶部商品图标出现后再继续后续动作。 |

模板来源：

| 模式 | 模板目录 | 颜色顺序 |
|---|---|---|
| 鼠标模式 | `assets/{file_name}` | RGB 截图转灰度 |
| ADB 模式 | `adb-assets/{file_name}` | BGR 截图转灰度 |

## 刷新流程参数

代码位置：`app/services/shop_flow_vision.py`

| 业务阶段 | 参数 | 当前值 | 说明 |
|---|---|---:|---|
| 商店刷新确认 | `SHOP_REFRESH_BUTTON_POINT` | `(237, 652)` | 点击“立即更新”。 |
| 商店刷新确认 | `SHOP_REFRESH_CONFIRM_POINT` | `(727, 474)` | 点击刷新确认。 |
| 刷新后加载判定 | `FIRST_SHOP_ITEM_MARKER_POINT` | `(1090, 109)` | 刷新后第一个商品行附近的蓝色标记参考点。 |
| 刷新后加载判定 | `FIRST_SHOP_ITEM_MARKER_COLOR` | `#24A7FD` | 第一个商品行蓝色标记颜色。 |
| 刷新后加载判定 | `SHOP_ITEM_MARKER_COLOR_TOLERANCE` | `24` | RGB 每个通道允许误差。 |
| 刷新后加载判定 | `SHOP_ITEM_MARKER_REGION_RADIUS` | `8` | 以参考点为中心的检测半径，会按截图分辨率缩放。 |
| 刷新后加载判定 | `SHOP_ITEM_MARKER_MIN_RATIO` | `0.12` | 检测区域内命中目标颜色的最低像素比例。 |
| 商店刷新确认 | 最大等待 | `10s` | 每 `0.2s` 截图检查一次，超时终止。 |
| 商店刷新确认 | 成功后等待 | `1s` | 商品行出现后再进入下一轮 OCR。 |

## 运行阶段状态

`scan_phase` 只影响价格 OCR 扫描哪些商品位：

| 阶段 | 何时设置 | 扫描商品位 |
|---|---|---|
| `top` | 进入商店后、刷新成功后 | `item_1` 到 `item_4` |
| `bottom` | 执行商店滑动后 | `item_5` 到 `item_6` |

价格 OCR 的统计触发点没有改变：backend 只有在 `buy_item()` 成功完成购买弹窗确认和顶部获得提示确认后返回，`RefreshEngine` 才会累计购买数量和金币消耗。
